# Security Policy

This repository contains guidance, not a deployable service. Security issues here usually fall into one of these categories:

- unsafe operational advice
- insecure coding examples
- recommendations that weaken release or access controls
- documentation that encourages leaking secrets or bypassing auditability

## Reporting

If you find a security problem in the guidance, open a private report with:

- the affected file
- the risky statement or example
- why it is unsafe
- a safer alternative

Do not open a public issue for material that could encourage insecure deployments before it is reviewed.

## Scope

This policy applies to:

- playbooks
- checklists
- templates

It does not guarantee response times or bug-bounty handling.
