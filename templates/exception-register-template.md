# Exception Register Template

```markdown
# Exception Register

| ID | Control Or Requirement | Exception Description | Business Reason | Risk Level | Compensating Controls | Owner | Approved By | Expiry / Review Date | Status |
|----|------------------------|-----------------------|-----------------|-----------|-----------------------|-------|-------------|----------------------|--------|
| EX-001 | Example control | Temporary gap | Delivery dependency | Medium | Manual review | Name | Name | YYYY-MM-DD | Open |
```

Rules:

- Every exception has an owner.
- Every exception has an expiry or review date.
- Expired exceptions must be closed, renewed, or escalated explicitly.
